
* 프로그램에서의 오류

- compile error:
  프로그램 코드 작성중 발생하는 문법적 오류
  대부분 detection 됨

- runtime error
  실행중인 프로그램이 의도하지 않은 동작(bug)를 하거나 프로그램이 중지되는 오류
  실행 오류는 비정상 종료가 되는 경우 시스템의 심각한 장애 발생 가능


* 예외 처리의 중요성

- 비정상 종료를 피하기 위하여
- 실행 오류가 발생한 경우 오류의 과정을 재현하는것은 현실적으로 힘듦
- 오류가 발생한 경우 log를 남겨서 log 분석을 통하여 bug 수정이 중요


* 오류와 예외 클래스

- System error : 가상 머신에서 발생, 프로그래머가 처리할수 없는 오류
  동적 메모리가 없는 경우, 스택 메모리 오버플로우 등

- Exception : 프로그램에서 제어할 수 있는 오류
  읽어들이려는 파일이 존재하지 않거나... 네트워크나 DB연결이 안되거나...

- 자바는 안전성이 중요한 언어로 대부분 프로그램에서 발생하는 오류에 대해 문법적으로 예외처리 해야함.

    Throwable
        |
Error   /   Exception



* 예외 클래스

java.lang.Object
  - java.lang.Throwable
    - java.lang.Exception
